# 快速原型

原型是**回答一个问题的一次性代码**。问题决定形态。

## 选择分支

识别要回答的问题：

- **"这个逻辑/状态模型对不对？"** → 构建一个小型交互式终端应用，把状态机推过那些纸上难以推理的情况。
- **"这个应该长什么样？"** → 在一个路由上生成多种风格迥异的 UI 变体，通过 URL 参数和浮动底栏切换。

如果模糊不清，默认选择与周围代码更匹配的分支（后端模块 → 逻辑；页面/组件 → UI）。

## 通用规则

1. **从第一天起就是一次性的，并清晰标注。** 放在将被使用的位置附近。命名让读者一眼看出是原型。
2. **一条命令即可运行。** 用项目已有的任务运行器。
3. **默认不持久化。** 状态只在内存中。
4. **跳过打磨。** 不写测试，不做超出"能跑"所需的错误处理。
5. **暴露状态。** 每次操作后，打印/渲染完整的相关状态。
6. **完成后删除或吸收。**

## 完成时

只有**答案**值得保留。把它连同所回答的问题一起记录在持久位置（commit 信息、ADR、issue 或 `NOTES.md`）。
