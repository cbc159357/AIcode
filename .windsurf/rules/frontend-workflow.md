# 前端页面开发流程 (Frontend Workflow)

新增前端页面或对现有页面进行大规模改造时，必须遵循「方案先行 → 文档确认 → 代码实施」的流程。

## 适用范围

以下场景触发本规则：

- 新建一个完整页面（view）
- 新建一个复杂组件（含多个子组件的组合体）
- 对现有页面进行布局级重构
- 涉及 ≥ 3 个文件的前端联动变更

> 微调样式、修 Bug、单组件内部优化**不触发**本规则。

## 流程

### Step 1 — 提供多方案供用户选择

在动手写代码之前，先向用户提供 **2~3 种设计方案**，每种方案需包含：

| 项目 | 必须 | 说明 |
|------|------|------|
| **方案名称** | ✅ | 一句话概括方案特征 |
| **布局草图** | ✅ | ASCII 或文字描述布局结构 |
| **组件拆分** | ✅ | 列出涉及的组件及职责 |
| **交互说明** | ✅ | 关键交互行为（点击/拖拽/切换等） |
| **优缺点** | ✅ | 每种方案的利弊对比 |
| **技术要点** | 可选 | 特殊技术选型或难点 |

方案之间应有**明显差异**（布局不同 / 交互模式不同 / 信息密度不同），而非仅配色或字号微调。

### Step 2 — 用户确认方案

- 等待用户明确选择某一方案（或要求组合/修改）。
- **禁止未经确认直接实施**。

### Step 3 — 生成设计文档

用户确认后，在 `docx/02_技术文档/前端/` 生成正式设计文档（遵循 doc-standards 规则）：

- 文件名：`YYYYMMDD_页面名称设计.md`
- 内容包含：确认的方案布局、组件清单、Props/Events 接口、Store 变更、路由变更、API 依赖
- 同步更新 `docx/README.md` 索引

### Step 4 — 实施代码

按照设计文档实施代码，遵循以下顺序：

1. **路由** — 注册新路由（如需要）
2. **Store** — 新增/修改状态管理（如需要）
3. **API 层** — 新增接口调用（如需要）
4. **页面组件** — 创建页面级 Vue SFC
5. **子组件** — 拆分提取子组件
6. **样式** — 全局 CSS 补充（如有新的公共样式）
7. **导航** — 更新侧栏/路由入口

### Step 5 — 构建验证

代码完成后执行 `npm run build`，确保零错误通过。

## 例外

以下情况可跳过多方案环节，但仍需先生成文档：

- 用户已给出**明确且详细**的布局和交互要求（无决策空间）
- 照搬已有页面模板仅改数据源
