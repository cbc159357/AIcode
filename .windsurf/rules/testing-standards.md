# 测试管理规范 (Testing Standards)

为了保证项目根目录与核心业务代码的整洁，所有临时测试脚本、API 验证文件以及阶段性实验脚本必须统一管理和约束。

## 1. 测试目录归档规范
- **统一目录**：所有测试文件、测试脚本、测试输出必须存放于项目母目录（即本项目的根目录 `d:/AI生成动漫/tests/`）下。
- **自动创建**：在编写或执行任何测试前，必须确保 `d:/AI生成动漫/tests/` 目录存在。若不存在，助手应当使用工具自动创建它。
- **严禁乱放**：严禁在 `pipeline/`、`server/`、`web/` 或新克隆的子项目（如 `Infinite-Canvas/`）根目录下直接丢弃临时测试脚本（如 `test_api.py`、`test.py` 等）。

## 2. 测试命名规范
- **Python 测试脚本**：以 `test_` 开头且具有描述性，例如 `test_manxiaobai_api.py`、`test_tts_synthesis.py`。
- **临时/实验性脚本**：以 `temp_` 或 `experiment_` 开头。
- **测试输出数据**：所有测试生成的临时图片、音频、JSON 响应等，必须输出到 `tests/test_output/` 文件夹中，严禁直接写入代码根目录。

## 3. 测试文件的代理与网络约束
- **科学上网/海外 API**：如果测试脚本需要请求海外服务（如 OpenAI、xAI/Grok、HuggingFace、Github 资源等），脚本中必须具备**代理兼容逻辑**：
  - 默认使用本地代理：`http://127.0.0.1:7890`（国内常用 Clash 端口）。
  - 必须提供 **Fallback 机制**（即代理连接失败时，自动切换为直连）。
  - 下载失败时提供直观的失败提示，并打印出原始 URL，方便用户在浏览器中手动下载。

## 4. 测试生命周期与安全约束
- **不污染主数据库/持久化目录**：测试脚本运行期间产生的任何临时状态，严禁写入主项目的 SQLite 数据库或正式的 `assets/` / `episodes/` 生产目录。
- **敏感信息脱敏**：
  - 严禁将含有明文 API Key、Token 或云端 SK 的测试脚本提交到 Git 仓库。
  - 在编写测试脚本时，优先读取环境变量（如 `os.getenv`）或外部配置文件；如若硬编码测试，在交付给用户前应当进行脱敏或主动提醒。
- **测试后清理**：
  - 鼓励在测试脚本中加入垃圾回收或临时文件清理逻辑。
  - 确保大文件（如测试视频、大批量图片）不被同步到代码托管库。

## 5. 新增测试步骤流程（助手行为准则）
1. 接收到生图/TTS/工作流测试需求。
2. 检查 `d:/AI生成动漫/tests/` 目录是否存在。如果不存在，立即创建。
3. 将测试脚本命名为 `tests/test_<描述>.py` 并写入其中。
4. 将脚本内的图片或媒体输出路径指向 `tests/test_output/`。
5. 运行测试并向用户汇报。
