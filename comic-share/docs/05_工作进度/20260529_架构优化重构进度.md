### 候选项 2 — 抽取 `workflow_store.py`（✅ 完成）

**问题**：`routers/runninghub.py` 路由层直接执行文件系统读写、JSON 序列化、工作流分析调用，违反关注点分离原则，路由函数体积臃肿。

**变更**：

- 新建 `backend/services/workflow_store.py`，封装所有文件 CRUD 操作：
  - `list_workflows()` — 遍历目录返回摘要列表
  - `get_workflow(id)` — 读取单个工作流，不存在抛 `FileNotFoundError`
  - `save_workflow(...)` — 创建/更新，内部调用 `workflow_analyzer` + `image_inferrer`
  - `update_image_inputs(id, inputs)` — 保存标注结果
  - `apply_model_overrides(id, overrides)` — 批量替换并重注元数据
  - `delete_workflow(id)` — 删除文件

- `routers/runninghub.py` 路由层变化：
  - 删除 `_DATA_DIR`, `_ensure_data_dir()`, 所有 `json.loads/writes`, `time.strftime` 调用
  - 删除 `import json`, `import time`, `from typing import ... Optional` 等已无用导入
  - 删除 `from core import workflow_analyzer as wa`, `from core import image_inferrer as ii`（职责下沉到 store）
  - 新增 `from services import workflow_store`
  - 每个路由处理器变为 1-4 行薄调用
