# 模型兼容检测页面设计

**创建日期**: 2026-05-28  
**路由**: `/workflow/model-check`  
**状态**: 待实施

---

## 一、功能目标

从已保存的 RunningHub 工作流中选取一个，展示其模型参数（按 modelGroups 分组），允许用户：
1. 查看每个模型槽位的当前值及其架构元数据
2. 通过 ModelSelector 搜索并替换模型
3. 查看替换后的兼容性报告（架构是否一致、量化格式等）
4. 将替换后的参数值持久化回工作流 JSON

---


## 四、数据流

```
listSavedWorkflows()          → 工作流下拉选项
getSavedWorkflow(id)          → RHWorkflowDetail (params + modelGroups)
suggestCompatibleModels(fn)   → 每个模型槽的推荐候选列表
applyModelOverrides(id, [...])→ PATCH /rh/workflows/{id}/models → 持久化
```

```ts
  path: 'workflow',

  children: [
```

## 六、新增后端端点

```
PATCH /api/v1/rh/workflows/{workflow_id}/models
Body: [{ nodeId, fieldName, value }]
→ 更新对应 param 的 currentValue，写回 JSON 文件
→ 返回 { ok, message }
```

## 七、RHWorkflowDetail 类型扩展

新增 `modelGroups?: ModelGroup[]` 字段，其中：

```ts
interface ModelGroup {
  groupId: string
  samplerNodeId: string
  params: RHWorkflowParam[]   // 该组内的模型类型参数
}
```
