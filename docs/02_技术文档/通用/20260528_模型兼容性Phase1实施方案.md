## 一、目标与范围

### 目标

用户在「模型配置」分组中修改任意模型字段（UNET / VAE / CLIP / LoRA）后：

1. **实时显示架构标签**：每个模型字段旁标注其所属架构族（`Flux2-Klein` / `SDXL` / `SD15` 等）
2. **检测跨族不一致**：同一 BFS 模型联动组内若出现不同架构族，高亮冲突节点
3. **内联推荐替换**：冲突字段展开下拉选择器，按正确类型（resourceType）+ 同一架构族（baseModel）过滤候选模型
4. **缓存本地管理**：Settings 页提供缓存刷新控制，支持手动 / 启动自动 / 定时三种策略

### 不做（Phase 2）

- 模型预设包（一键原子替换整组）
- 触发词（triggerWords）自动注入提示词
- 用户私有上传模型的元数据管理

## 二、整体架构

```
RunningHub API
/openapi/v2/resource/list
        │
        ▼
model_cache_service.py ──写──▶ data/runninghub/model_cache.json
        │
        ├─ load_cache()          ← 每次请求时内存热读
        ├─ get_model_meta(filename) → ModelMeta
        ├─ search_models(baseModel, resourceType, q) → list[ModelMeta]
        └─ normalize_base_model(raw) → str  (归一化架构族名)

runninghub_client.py
        ├─ find_model_groups(raw_nodes) → list[ModelGroup]  (BFS 联动组)
        ├─ annotate_params_with_meta(params, cache) → list  (附加 modelMeta)
        └─ save_workflow() → 调用以上两函数，写入 paramGroups + modelMeta

routers/runninghub.py
        ├─ GET  /rh/model-cache/status
        ├─ POST /rh/model-cache/refresh
        ├─ GET  /rh/models/suggest?filename=&resourceType=&baseModel=
        └─ GET  /rh/models/search?q=&baseModel=&resourceType=

前端
        ├─ ParamGroup.vue  — 架构标签显示 + 失焦检测 + 冲突高亮
        ├─ ModelSelector.vue (新建) — 下拉 + 手动搜索 + 实时过滤
        ├─ WorkflowCard.vue — 加载时传入 modelMeta
        └─ SettingsView.vue — 缓存管理卡片
```

## 四、后端实现

### 4.1 新建 `services/model_cache_service.py`

```python
"""
RunningHub 模型元数据缓存服务。
分页拉取 /openapi/v2/resource/list，构建 byFileName 索引，本地 JSON 持久化。
"""
from __future__ import annotations

import json
import re
from datetime import datetime, timezone
from pathlib import Path
from typing import Any

import httpx

from core.config import get_config

_CACHE_PATH = Path("data/runninghub/model_cache.json")
_RH_RESOURCE_URL = "https://www.runninghub.cn/openapi/v2/resource/list"

_BASE_MODEL_NORMALIZE: dict[str, str] = { ... }  # 见 3.3

_QUANT_PATTERNS: list[tuple[re.Pattern, str]] = [
    (re.compile(r"fp8", re.I), "fp8"),
    (re.compile(r"int8", re.I), "int8"),
    (re.compile(r"gguf", re.I), "gguf"),
    (re.compile(r"nf4", re.I), "nf4"),
    (re.compile(r"bf16", re.I), "bf16"),
]

_cache_data: dict = {}  # 内存热缓存


def _infer_quantization(filename: str) -> str:
    for pattern, quant in _QUANT_PATTERNS:
        if pattern.search(filename):
            return quant
    return "fp16"


def normalize_base_model(raw: str) -> str:
    return _BASE_MODEL_NORMALIZE.get(raw, raw or "unknown")


def _parse_record(record: dict) -> tuple[str, dict]:
    """将 API 单条记录转换为 (filename, meta) 对。"""
    filename = record.get("nodeModelName", "")
    versions = record.get("versions", [])
    base_model_raw = versions[0].get("baseModel", "") if versions else ""
    trigger_words = versions[0].get("triggerWords", "") if versions else ""
    tags = [t["name"] for t in record.get("tags", []) if isinstance(t, dict)]
    return filename, {
        "resourceType": record.get("resourceType", ""),
        "baseModel": normalize_base_model(base_model_raw),
        "baseModelRaw": base_model_raw,
        "resourceName": record.get("resourceName", ""),
        "quantization": _infer_quantization(filename),
        "tags": tags,
        "triggerWords": trigger_words,
    }


async def fetch_all_models(api_key: str) -> dict[str, Any]:
    """
    分页拉取全量模型列表，构建 byFileName 索引。
    约 114 页 × 10 条 = 1135 条模型。
    """
    headers = {"Authorization": f"Bearer {api_key}"}
    by_filename: dict[str, dict] = {}
    page = 1

    async with httpx.AsyncClient(timeout=30) as client:
        while True:
            resp = await client.post(
                _RH_RESOURCE_URL,
                headers=headers,
                json={"current": page, "size": 100},  # 每页 100 条，减少请求次数
            )
            resp.raise_for_status()
            data = resp.json().get("data", {})
            for record in data.get("records", []):
                filename, meta = _parse_record(record)
                if filename:
                    by_filename[filename] = meta
            if not data.get("hasNext", False):
                break
            page += 1

    return {
        "fetchedAt": datetime.now(timezone.utc).isoformat(),
        "totalCount": len(by_filename),
        "byFileName": by_filename,
    }


async def refresh_cache(api_key: str) -> dict:
    """拉取并覆盖写入本地缓存，返回缓存摘要。"""
    global _cache_data
    _CACHE_PATH.parent.mkdir(parents=True, exist_ok=True)
    data = await fetch_all_models(api_key)
    _CACHE_PATH.write_text(json.dumps(data, ensure_ascii=False, indent=2), encoding="utf-8")
    _cache_data = data
    return {"totalCount": data["totalCount"], "fetchedAt": data["fetchedAt"]}


def load_cache() -> dict[str, Any]:
    """读取本地缓存（内存热缓存优先）。"""
    global _cache_data
    if _cache_data:
        return _cache_data
    if _CACHE_PATH.exists():
        _cache_data = json.loads(_CACHE_PATH.read_text(encoding="utf-8"))
    return _cache_data


def get_model_meta(filename: str) -> dict | None:
    return load_cache().get("byFileName", {}).get(filename)


def get_cache_status() -> dict:
    if not _CACHE_PATH.exists():
        return {"exists": False, "totalCount": 0, "fetchedAt": None}
    cache = load_cache()
    return {
        "exists": True,
        "totalCount": cache.get("totalCount", 0),
        "fetchedAt": cache.get("fetchedAt"),
    }


def search_models(
    base_model: str | None = None,
    resource_type: str | None = None,
    q: str | None = None,
    limit: int = 50,
) -> list[dict]:
    """
    在缓存中搜索模型。
    base_model: 归一化后的架构族（精确匹配）
    resource_type: UNET / CLIP / VAE / LORA（精确匹配）
    q: 文件名或 resourceName 模糊搜索
    """
    by_filename = load_cache().get("byFileName", {})
    results = []
    q_lower = q.lower() if q else None

    for filename, meta in by_filename.items():
        if base_model and meta.get("baseModel") != base_model:
            continue
        if resource_type and meta.get("resourceType", "").upper() != resource_type.upper():
            continue
        if q_lower and q_lower not in filename.lower() and q_lower not in meta.get("resourceName", "").lower():
            continue
        results.append({"filename": filename, **meta})
        if len(results) >= limit:
            break

    return results
```

### 4.2 修改 `runninghub_client.py`

#### 新增函数：`find_model_groups(raw_nodes)`

复用已有 `_build_downstream_map`，沿 MODEL / CLIP / VAE 边反向追踪到 KSampler，将上游模型节点归为联动组。

```python
_SAMPLER_TYPES = {"KSampler", "KSamplerAdvanced", "SamplerCustom", "UnetSchedulerStepsLoader"}
_MODEL_EDGE_TYPES = {"MODEL", "CLIP", "VAE", "CONDITIONING"}

def find_model_groups(raw_nodes: dict) -> list[dict]:
    """
    BFS 识别"模型联动组"：
    同一 KSampler 的所有上游 Loader 节点归为一组。
    返回: [{"groupName": str, "samplerNodeId": str, "modelNodeIds": [str]}]
    """
    # 构建 上游 → 下游 映射（已有基础设施）
    downstream_map = _build_downstream_map(raw_nodes)
    groups = []

    for node_id, node_data in raw_nodes.items():
        class_type = node_data.get("class_type", "")
        if class_type not in _SAMPLER_TYPES:
            continue
        # 向上反向 BFS，找所有通过 MODEL/CLIP/VAE 边连入的 Loader 节点
        model_node_ids = _collect_upstream_loaders(str(node_id), raw_nodes)
        if model_node_ids:
            groups.append({
                "groupName": f"采样节点 #{node_id} 模型组",
                "samplerNodeId": str(node_id),
                "modelNodeIds": model_node_ids,
            })

    return groups


def _collect_upstream_loaders(sampler_id: str, raw_nodes: dict) -> list[str]:
    """反向 BFS：从 KSampler 向上找所有 Loader 节点。"""
    loader_keywords = ("loader", "checkpoint", "lora")
    visited = set()
    queue = [sampler_id]
    loaders = []

    # 构建 下游 → 上游 映射
    upstream_map: dict[str, list[str]] = {}
    for node_id, node_data in raw_nodes.items():
        for v in node_data.get("inputs", {}).values():
            if isinstance(v, list) and len(v) == 2:
                upstream_id = str(v[0])
                downstream_id = str(node_id)
                upstream_map.setdefault(downstream_id, []).append(upstream_id)

    while queue:
        nid = queue.pop(0)
        if nid in visited:
            continue
        visited.add(nid)
        node = raw_nodes.get(nid) or raw_nodes.get(int(nid), {})
        ct = node.get("class_type", "").lower()
        if any(k in ct for k in loader_keywords):
            loaders.append(nid)
        for upstream in upstream_map.get(nid, []):
            queue.append(upstream)

    return loaders
```

### 5.2 新建 `components/workflow/ModelSelector.vue`

**职责**：单个模型字段的下拉搜索选择器，当 `incompatible=true` 时自动展开并高亮。

**Props**：

| Prop | 类型 | 说明 |
|------|------|------|
| `modelValue` | `string` | 当前文件名值 |
| `nodeId` | `string` | 节点 ID（用于 emit） |
| `fieldName` | `string` | 字段名 |
| `resourceType` | `string` | 节点类型（CLIP / VAE / LORA 等） |
| `currentBaseModel` | `string \| undefined` | 当前值的 baseModel |
| `targetBaseModel` | `string \| undefined` | 目标 baseModel（其他节点决定的族） |
| `incompatible` | `boolean` | 是否处于不兼容状态（控制展开） |

**交互流程**：

```
incompatible = true
  → 输入框边框变红 + 下方自动展开候选列表
  → 调用 searchModels({ baseModel: targetBaseModel, resourceType }) 预载候选
  → 用户可手动在输入框输入关键词过滤（debounce 300ms）
  → 点击候选项 → emit('update:modelValue', filename)
  → incompatible 由父组件重新计算后恢复 false → 收起
```

**候选列表展示格式**：

```
┌────────────────────────────────────────┐
│ 🔍 [搜索框，已预填 targetBaseModel 关键词] │
├────────────────────────────────────────┤
│ [Flux2-Klein] realDream_Klein           │  ← resourceName
│ realDream_klein9BV1.safetensors  fp16  │  ← filename + quantization
├────────────────────────────────────────┤
│ [Flux2-Klein] T5-XXL FP8              │
│ t5xxl_fp8_e4m3fn.safetensors    fp8   │
└────────────────────────────────────────┘
```

#### 新增：失焦实时兼容性检测

在 `nodeGroups` 模型组内，维护 `groupBaseModel`（BFS 组内权威 baseModel，取 UNET 的）：

```typescript
/** 整个 ParamGroup 内，取 UNET 节点的 baseModel 作为权威架构 */
const groupBaseModel = computed(() => {
  for (const ng of nodeGroups.value) {
    for (const p of ng.params) {
      if (p.modelMeta?.resourceType === 'UNET') return p.modelMeta.baseModel
    }
  }
  return null
})

/** 检查某个 param 是否与组内权威架构冲突 */
function isIncompatible(param: RHWorkflowParam): boolean {
  if (!groupBaseModel.value || !param.modelMeta) return false
  if (param.modelMeta.resourceType === 'UNET') return false   // UNET 是权威，不与自己比
  if (param.modelMeta.resourceType === 'LORA') return false   // LoRA 推荐但不报错
  return param.modelMeta.baseModel !== groupBaseModel.value
}
```

#### 修改：模型字段渲染

## 十、模型变更影响域分析

> 结论先行：**影响范围可控，不属于连锁灾难**。CLIP/VAE 是硬性断路，其余（采样参数、插件节点）是软性降质。Phase 1 处理硬性断路即可覆盖绝大多数真实错误。

### 10.1 影响分级

| 级别 | 描述 | 是否报错 | 示例 |
|------|------|---------|------|
| **L0 — 直接断路** | 架构不匹配导致 RunningHub 任务直接失败 | ✅ 报错 | Flux UNET + SD1.5 CLIP |
| **L1 — 质量降级** | 任务能跑完但输出质量差（参数数值范围不适配） | ❌ 不报错 | Flux 工作流 cfg=1 换成 SD1.5 后 cfg 应调为 7 |
| **L2 — 节点失效** | 某些架构专用节点静默失效或被绕过 | ❌ 不报错 | 在 SDXL 工作流中遗留 `FluxGuidance` 节点 |
| **L3 — 插件不兼容** | ControlNet / IPAdapter 模型架构不匹配 | ✅ 报错（运行时） | SD1.5 ControlNet 注入 SDXL 工作流 |

### 10.2 L0 直接断路（必须处理）

这是 Phase 1 的核心目标：

| 修改的节点 | 必须同步更换 | 原因 |
|-----------|------------|------|
| UNET（任意架构变更） | CLIPLoader 全部字段 | CLIP 编码器维度与 UNET attention 层强绑定 |
| UNET（Flux → SD1.5/SDXL） | VAELoader | Flux 使用专用 AE（16-channel），标准 VAE 是 4-channel，格式不兼容 |
| UNET（SD1.5 → SDXL） | VAELoader | 可选但强烈建议（SDXL 有专用 VAE，通用 VAE 也部分可用） |
| CheckpointLoader（含 UNET+CLIP+VAE） | 无需单独换 | Checkpoint 自带完整三件套 |

**CLIP 字段对应关系**（不同架构所需 CLIP 不同）：

| 架构族 | clip_name1 | clip_name2 |
|--------|-----------|-----------|
| SD 1.5 | clip-vit-l | — |
| SDXL | clip_g | clip_l |
| Flux.1 | t5xxl_fp8_e4m3fn | clip_l |
| Flux2-Klein | t5xxl_fp8_e4m3fn | clip_l（或同族 CLIP） |
| HiDream | 4 个 CLIP（专用节点） | — |

### 10.3 L1 质量降级（Phase 1 警告，Phase 2 推荐）

**KSampler 关键参数对照**：

| 参数 | Flux系 | SD1.5 | SDXL |
|------|--------|-------|------|
| `cfg` | 1.0（FluxGuidance 单独控） | 7～9 | 6～8 |
| `steps` | schnell: 4, dev: 20～30 | 20～30 | 25～35 |
| `sampler_name` | euler / dpmpp_2m | euler_a / dpm++2m | euler_a / dpm++2m |
| `scheduler` | simple / sgm_uniform | karras / exponential | karras |
| `denoise` | 1.0（全去噪） | 0.7～1.0 | 0.7～1.0 |

> **结论**：KSampler 参数不匹配**不会报错**，只会出图效果差（如 Flux 工作流 cfg=1 切换到 SD1.5 后图像对比度低）。Phase 1 可以弹出"建议调整采样参数"的软提示，但不强制。

**EmptyLatentImage 分辨率**：

| 架构族 | 最优原生分辨率 |
|--------|------------|
| SD 1.5 | 512×512 |
| SDXL | 1024×1024 |
| Flux | 任意（通常 1024×1024） |

切换架构时若分辨率维持原值，**不会报错**，只影响出图质量。

### 10.4 L2 架构专用节点静默失效

这类节点在工作流中"占位"，切换架构后会被忽略或产生无意义输出：

| 节点 | 归属架构 | 切换后行为 |
|------|---------|----------|
| `FluxGuidance` | Flux 专用 | 传给非 Flux UNET 时参数被忽略（不报错） |
| `ModelSamplingFlux` | Flux 专用 | 非 Flux 时无效果 |
| `CLIPSetLastLayer`（clip_skip=2） | SD1.5 习惯 | SDXL/Flux 下 clip_skip 通常应为 1，设 2 会导致语义理解偏差 |
| `BasicGuider` / `CFGGuider` | Flux 路由方式 | 接 SD1.5 UNET 时行为未定义 |

> **结论**：L2 影响存在但用户无感知（不报错只是不对）。Phase 1 可在检测到跨架构时附一条 **"工作流含架构专用节点，建议使用同架构工作流"** 的提示，但不阻断操作。

### 10.5 L3 插件节点（影响范围评估）

| 插件类型 | 架构绑定性 | Phase 1 处理建议 |
|---------|-----------|----------------|
| ControlNet（控制网络模型） | **强绑定**：sd15/sdxl/flux 各一套 | 检测到 ControlNetLoader 时，将其列入 L0 告警 |
| IPAdapter（参考图风格迁移） | **强绑定**：同上 | 同上 |
| FaceDetailer / ADetailer | 弱绑定（内部调用独立采样器） | 一般无需处理 |
| Upscale 模型（ESRGAN 等） | **无绑定** | 不受影响 |
| 分割 / 遮罩节点 | **无绑定** | 不受影响 |
| 视频节点（Wan2.x i2v） | **完全独立架构** | 不在同一 BFS 模型组内，自然隔离 |

**ControlNet / IPAdapter 检测方案（Phase 1 扩展）**：

在 `find_model_groups()` 中，扩展 loader 关键词识别：
```python
_PLUGIN_LOADER_TYPES = {
    "ControlNetLoader", "ControlNetApply", "ControlNetApplyAdvanced",
    "IPAdapterModelLoader", "IPAdapterApply",
}
# 发现这些节点时，在 modelGroup 的 modelNodes 中附加标记 "pluginType": "controlnet"
# 前端对这类节点显示额外警告："ControlNet 模型需与主模型架构匹配"
```

### 10.6 影响域总结图

```
修改 UNET 架构（如 Flux → SDXL）
│
├─── L0 直接断路 [必须处理，Phase 1 核心]
│    ├── CLIPLoader（全字段）      ← 强制更换
│    └── VAELoader                ← 强制更换
│
├─── L1 质量降级 [软提示]
│    ├── KSampler.cfg             ← 建议调整
│    ├── KSampler.sampler_name    ← 建议调整
│    ├── KSampler.scheduler       ← 建议调整
│    └── EmptyLatentImage 分辨率  ← 建议调整
│
├─── L2 专用节点失效 [信息提示]
│    ├── FluxGuidance             ← 提示"此节点 Flux 专用"
│    └── CLIPSetLastLayer         ← 提示"clip_skip 建议改为 1"
│
└─── L3 插件节点 [按需处理]
     ├── ControlNetLoader         ← 纳入 L0 告警范围
     ├── IPAdapterModelLoader     ← 纳入 L0 告警范围
     └── 其余（Upscale/分割）     ← 不受影响，忽略
```

### 10.7 对 Phase 1 实施策略的影响

- **BFS 模型组范围**：默认扩展到包含 ControlNet/IPAdapter Loader 节点（纳入 L0 告警）
- **采样参数**：检测到架构变更时，在模型配置组底部追加一个**"采样参数参考"**折叠区，列出该架构推荐的 cfg / steps / sampler 值（只读参考，不强制替换）
- **专用节点提示**：工作流中若存在 `FluxGuidance`、`ModelSamplingFlux` 等节点，在架构不一致时显示一行提示

## 十一、本地跑通配置匹配策略（经验预设）

> **核心思想**：与其依赖 RH 官方 API 提供的"理论兼容性"，不如从**用户过去成功运行的记录**中直接提取"实测兼容配置"。经验预设的可信度 100%，且能捕获采样参数、分辨率等 API 无法覆盖的细节。

### 11.1 优先级体系

```
推荐来源优先级（由高到低）:

1. ✅ 本地经验预设   — 该工作流曾跑通的完整参数快照（本章）
2. 📋 用户手动预设   — Phase 2：用户自己创建并保存的预设包
3. 🔍 RH API 理论推荐 — 同 baseModel + 同 resourceType（第十章 Phase 1 方案）
4. ❓ 未知           — 模型不在缓存中，只显示"未知架构"标签
```

### 11.2 数据采集：任务成功时自动快照

**触发时机**：`TaskRunner.vue` 轮询到任务状态 `success` 时，记录本次提交的完整参数快照。

**存储路径**：`data/runninghub/proven_configs/{workflowId}.json`

**数据结构**：

```json
{
  "workflowId": "7452d2c4-ab26-4e78-9d2e-9eea81cbc983",
  "configs": [
    {
      "id": "cfg_20260528_143012",
      "timestamp": "2026-05-28T14:30:12Z",
      "taskId": "rh_task_xyz",
      "label": "用户自定义备注（可选）",
      "params": {
        "4":  { "unet_name": "realDream_klein9BV1.safetensors" },
        "11": { "clip_name1": "t5xxl_fp8_e4m3fn.safetensors",
                "clip_name2": "clip_l.safetensors" },
        "10": { "vae_name": "ae.safetensors" },
        "3":  { "steps": 20, "cfg": 1.0,
                "sampler_name": "euler", "scheduler": "simple",
                "denoise": 1.0 },
        "6":  { "text": "1girl, anime style..." }
      },
      "modelFingerprint": "realDream_klein9BV1.safetensors|t5xxl_fp8_e4m3fn.safetensors|ae.safetensors"
    }
  ]
}
```

**`modelFingerprint`**：所有模型字段值拼接的指纹字符串，用于快速判断"当前参数是否与某个历史配置完全一致"（去重用）。

### 11.3 匹配算法

当用户修改任意模型字段时，触发以下匹配逻辑：

```
输入: 当前工作流 workflowId + 刚修改的字段 (nodeId, fieldName, newValue)

Step 1: 加载 proven_configs/{workflowId}.json
Step 2: 遍历所有历史配置，找出 params 中包含该字段值的配置
        匹配条件: config.params[nodeId][fieldName] == newValue
Step 3: 命中的配置按 timestamp 倒序排列（最近优先）
Step 4: 取前 3 条作为推荐候选，返回给前端
```

**匹配结果示例**：

```
用户将 unet_name 改为 "realDream_klein9BV1.safetensors"
→ 命中历史配置 cfg_20260528_143012
→ 前端显示：

  ┌─ 🟢 曾经跑通的配置 ─────────────────────────────────┐
  │  2026-05-28 14:30  (realDream_Klein 套装)            │
  │                                                      │
  │  会同步设置:                                          │
  │  CLIP: t5xxl_fp8_e4m3fn + clip_l                    │
  │  VAE:  ae.safetensors                                │
  │  步数: 20  CFG: 1.0  采样器: euler/simple            │
  │                                                      │
  │  [✅ 一键应用全部]   [仅应用模型组]   [忽略]          │
  └──────────────────────────────────────────────────────┘
```

### 11.4 应用策略（三档）

| 档位 | 操作 | 替换范围 |
|------|------|---------|
| **一键应用全部** | 将历史配置的所有参数覆盖当前值 | 所有节点参数（含采样参数、分辨率、提示词除外） |
| **仅应用模型组** | 只替换 BFS 模型组内的 Loader 字段 | UNET / CLIP / VAE / LoRA 文件名 |
| **忽略** | 关闭推荐，不做任何更改 | — |

> **注意**：默认不替换提示词（`CLIPTextEncode.text`），避免用户辛苦写的提示词被覆盖。

### 11.7 经验预设的局限与规避

| 局限 | 规避策略 |
