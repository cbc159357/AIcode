# 工作流参数分组显示设计

> 日期: 2026-05-28  
> 状态: 已决策，待实施  
> 依赖: `20260528_工作流输入端语义分析.md`（imageInputs 已实现）

---

## 背景与目标

`analyzedParams` 当前是一个按 priority 排序的平铺列表，用户在复杂工作流（30+ 参数）中难以快速定位需要修改的内容。本方案在 imageInputs 单独区块之外，对 `analyzedParams` 实现**启发式功能分组 + 折叠展示**。

---

## 决策摘要

| 维度 | 决策 |
|------|------|
| 数据来源 | 启发式推断（纯 class_type 规则，不依赖完整工作流 JSON） |
| imageInputs 位置 | 保持独立，仍在 WorkflowCard 顶部单独渲染 |
| 默认折叠行为 | 「提示词」「采样控制」「分辨率」默认展开；「模型配置」「输出设置」默认折叠 |
| 数据存储 | 后端在 `save_workflow` 时同步生成 `paramGroups` 字段存入 JSON |
| 前端架构 | `ParamEditor.vue` 接受 `paramGroups`，抽取 `ParamGroup.vue` 子组件 |

---

## 数据结构

### `paramGroups` 字段（存入工作流 JSON）

```typescript
interface ParamGroup {
  groupId: string           // 唯一标识，如 "prompt" | "sampling" | "resolution" | "model" | "output" | "other"
  title: string             // 展示标题，如 "提示词"
  defaultCollapsed: boolean // 初始折叠状态
  nodeIds: string[]         // 本组包含的节点 ID 列表
}
```

**示例（`2058857307617980417.json`）：**

```json
"paramGroups": [
  { "groupId": "prompt",     "title": "提示词",   "defaultCollapsed": false, "nodeIds": ["12","91","93","96","99"] },
  { "groupId": "sampling",   "title": "采样控制", "defaultCollapsed": false, "nodeIds": ["63"] },
  { "groupId": "resolution", "title": "分辨率",   "defaultCollapsed": false, "nodeIds": ["28","97","98"] },
  { "groupId": "model",      "title": "模型配置", "defaultCollapsed": true,  "nodeIds": ["15","44","45"] },
  { "groupId": "output",     "title": "输出设置", "defaultCollapsed": true,  "nodeIds": ["86"] },
  { "groupId": "other",      "title": "其他参数", "defaultCollapsed": false, "nodeIds": ["84","89","90"] }
]
```

> `paramGroups` 仅含节点 ID，参数的实际值仍从 `analyzedParams` 按 `nodeId` 查找。
> `analyzedParams` 保持不变（向后兼容）。

## 分组推断规则

后端新增函数 `group_params_by_function(nodes_json, analyzed_params)` 实现以下映射：

| groupId | defaultCollapsed | 匹配 class_type（白名单） |
|---------|-----------------|--------------------------|
| `prompt` | false | `CLIPTextEncode`, `JjkText`, `CR Prompt Text`, `PrimitiveStringMultiline`, `RH_LLMAPI_Pro_Node` |
| `sampling` | false | `KSampler`, `KSamplerAdvanced`, `SamplerCustom`, `Seed (rgthree)`, `FluxGuidance` |
| `resolution` | false | `EmptyLatentImage`, `EmptySD3LatentImage`, `TTResolutionSelector` |
| `model` | true | `UNETLoader`, `CLIPLoader`, `VAELoader`, `CheckpointLoaderSimple`, `LoraLoader`, `LoraLoaderModelOnly`, `DualCLIPLoader`, `TripleCLIPLoader` |
| `output` | true | `SaveImage`, `PreviewImage`, `VHS_VideoCombine`, `easy imageColorMatch` |
| `other` | false | 不属于以上任何组的剩余节点 |

**额外规则：**
- `imageInputs` 中已有的 `nodeIds` 不会出现在 `paramGroups` 里（双方互斥）
- 节点没有任何 `analyzedParams` 时，不生成对应分组
- `other` 组若为空，则不生成该条目

## 组件架构

```
WorkflowCard.vue
├── [图片输入区]          ← 现有，imageInputs 驱动，不变
├── ParamEditor.vue       ← 重构：接受 paramGroups + params
│   └── ParamGroup.vue    ← 新建：渲染单个折叠组
│       └── [参数行]      ← 现有参数行逻辑提取到此处
└── TaskRunner.vue        ← 不变
```

### `ParamGroup.vue`（新建，≤120 行）

**Props：**

```typescript
defineProps<{
  group: ParamGroup
  params: RHWorkflowParam[]   // 本组过滤后的参数子集
}>()
```

**Emits：**

```typescript
emit('update', nodeId: string, fieldName: string, value: string | number | boolean)
```

**行为：**
- 标题栏：`group.title` + 折叠箭头 + 参数数量 badge
- 初始折叠状态由 `group.defaultCollapsed` 控制
- 内部渲染参数行（从 ParamEditor 提取现有逻辑）

### `ParamEditor.vue`（重构，≤100 行）

**Props 变更：**

```typescript
defineProps<{
  params: RHWorkflowParam[]
  paramGroups?: ParamGroup[]          // 新增，有则按组渲染
  excludeNodeIds?: string[]           // 保留
}>()
```

**渲染逻辑：**
- 有 `paramGroups`：遍历组，每组渲染一个 `ParamGroup`
- 无 `paramGroups`：保持现有平铺列表渲染（向后兼容）
