# 任务媒体管理页面设计

> 创建日期: 2026-05-28
> 状态: 设计阶段
> 路由: `/tasks`

---

## 一、需求描述

用户在工作台每次执行 pipeline（生成剧本 → 生图 → TTS → 合成）会产出多种媒体资源。
需要一个统一的页面来**浏览、归档、管理**这些生成产物。

### 核心诉求

1. 生成内容按**用户命名的归档**分组（如"第一集"、"恐怖篇"等）
2. 归档通过**下拉框**选择/创建/重命名
3. 每个归档下包含多个**子任务**（每次 pipeline 执行的产物）
4. 左侧提供**树形结构**进行切换导航
5. 点击子任务后显示其对应的**媒体资源**（图片、音频、视频）

### 三层数据模型

| 层级 | 概念 | 说明 |
|------|------|------|
| L1 | **归档 (Archive)** | 用户命名的顶层分组，下拉框选择/新建 |
| L2 | **子任务 (Task)** | 一次 pipeline 执行，自动命名（时间戳 + 序号） |
| L3 | **媒体资源 (Media)** | 图片/音频/视频文件，属于某个子任务的产出 |

## 四、后端 API 设计

### 路由前缀: `/api/v1/archives`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 列出所有归档 |
| POST | `/` | 新建归档 `{ name }` |
| PUT | `/{archive_id}` | 重命名归档 |
| DELETE | `/{archive_id}` | 删除归档（含所有子任务） |
| GET | `/{archive_id}/tasks` | 列出该归档下所有子任务 |
| POST | `/{archive_id}/tasks` | 创建子任务（pipeline 开始时自动调用） |
| GET | `/{archive_id}/tasks/{task_id}` | 子任务详情 + 媒体列表 |
| DELETE | `/{archive_id}/tasks/{task_id}` | 删除子任务 |
| GET | `/{archive_id}/tasks/{task_id}/media` | 媒体文件列表（含类型/缩略图URL） |
| GET | `/media/{file_path}` | 静态文件服务（图片/音频/视频） |

### 响应格式

```json
// GET /api/v1/archives
{
  "archives": [
    { "id": "arc_001", "name": "第一集", "task_count": 3, "updated_at": "..." }
  ]
}

// GET /api/v1/archives/{id}/tasks/{tid}/media
{
  "media": [
    { "type": "image", "filename": "scene_01.png", "url": "/api/v1/archives/media/...", "size": 204800 },
    { "type": "audio", "filename": "narration_01.wav", "url": "...", "duration": 5.2 },
    { "type": "video", "filename": "final.mp4", "url": "...", "duration": 120 }
  ]
}
```

## 七、交互细节

1. **归档下拉** — 顶部 `a-select` 组件，支持搜索 + 新建（输入不存在的名称自动创建）
2. **树形导航** — `a-tree` 组件，归档为父节点，子任务为叶节点
3. **媒体画廊** — 网格布局，顶部 Tab 筛选（全部/图片/音频/视频）
4. **媒体预览** — 图片点击放大 lightbox，音频内联播放，视频打开右抽屉预览
5. **批量操作** — 多选后可批量删除、导出
6. **自动归档** — 工作台 pipeline 执行时，自动将产物存入当前选中的归档

## 八、与现有系统的关系

| 现有概念 | 新概念 | 关系 |
|----------|--------|------|
| Project | — | 暂保留，归档独立于 project |
| Episode | Archive | 功能类似但归档更轻量，不绑定剧本 |
| Pipeline 执行 | Task | 一次 pipeline = 一个 task |
| 生图/TTS/视频产出 | Media | 归集到 task 下统一管理 |

### Pipeline 集成点

`usePipeline.ts` 在 pipeline 开始时：
1. 检查当前选中的归档（若无则创建默认归档）
2. `POST /archives/{id}/tasks` 创建子任务
3. 各步骤产出文件写入该 task 目录
4. Pipeline 完成后更新 task status
